# ✅ Чек-лист проверки приложения «Код Мастер»

## 1. Запуск и главное окно
- [x] Приложение запускается на Python 3.12+ без ошибок импорта
- [x] Главное окно 800×600, можно развернуть, минимальный размер 640×480
- [x] Верхняя панель: логотип, индикатор порта, кнопки «Настройки» и «Выход»
- [x] Горизонтальное меню с 5 карточками (Поиск, Триггеры, Шлюз, Библиотека, Прошивка), переключение страниц
- [x] Центральная область QStackedWidget с 5 страницами
- [x] Нижняя статус-строка с информацией
- [x] Тёмная тема применяется глобально, цвета соответствуют StarLine Master
- [x] Переключатель светлой/тёмной темы
- [x] Кнопка «📄 Логи» открывает папку с логами

## 2. Модальное окно выбора COM-порта
- [x] Открывается при нажатии «Настройки», если порт не выбран
- [x] Список портов, выбор скорости, чекбокс эмуляции
- [x] Кнопка «Подключить»: успех/ошибка, сохранение в Config

## 3. Страница «Поиск» (CAN мониторинг)
- [x] Два канала CAN1/CAN2 в QSplitter
- [x] Кнопки Старт/Стоп/Очистить, фильтр ID, чекбокс «Приостановить"
- [x] Таблица с временем, ID, D0–D7, DLC, лимит 50 000 строк, чередование фона
- [x] Панель ручной отправки: ID + 8 Data, подсветка отправленных пакетов
- [x] Статистика «Принято: N | Скорость: N пак/с»
- [x] Контекстное меню: копировать ID/данные, создать триггер из пакета
- [x] Циклическая отправка с интервалом
- [x] Запись в CSV
- [x] Подсветка активности каналов (зелёный/красный)
- [x] Глобальные кнопки «Запустить оба», «Остановить оба», «Очистить всё»

## 4. Страница «Триггеры»
- [x] 10 блоков триггеров с полями приёма/отправки ID и D0–D7
- [x] Чекбокс «Активен», поле задержки ответа
- [x] Кнопка «Тест», счётчик срабатываний
- [x] Логика: совпадение → автоматический ответ в тот же канал с учётом задержки
- [x] Кнопки «Применить триггеры» / «Остановить"
- [x] Сохранение/загрузка триггеров отдельно

## 5. Страница «Шлюз»
- [x] Секция «Игнорировать» (10 ID), чекбокс включения
- [x] 10 правил подмены с направлением (CAN1↔CAN2)
- [x] Кнопки «Запустить шлюз» / «Остановить"
- [x] Логика ретрансляции с фильтрацией и подменой
- [x] Сохранение/загрузка правил отдельно

## 6. Страница «Прошивка»
- [x] Выбор файла .bin/.hex, кнопка «Обзор»
- [x] Кнопка «Инфо» для диагностики bootloader
- [x] Прогресс-бар, кнопки «Загрузить» и «Проверить»
- [x] Фоновый поток, обновление прогресса
- [x] Реализован протокол AN3155: синхронизация, стирание, запись с повторами

## 7. Протокол CAN-кадров
- [x] Упаковка/распаковка с маркерами 0xBB/0xAA, контрольной суммой XOR
- [x] Поддержка стандартного (11 бит) и расширенного (29 бит) ID

## 8. Многопоточность и SerialManager
- [x] Чтение COM-порта в отдельном QThread, сигналы new_can_frame, error_occurred, heartbeat
- [x] Автоматическое переподключение при обрыве (с настройкой)
- [x] Эмулятор FakeSerial с настройкой процента ошибок

## 9. Настройки и логирование
- [x] Config (синглтон), сохранение/загрузка config.json
- [x] Экспорт/импорт профилей (триггеры, шлюз)
- [x] Логирование с ротацией (1 МБ, 3 файла)
- [x] Логируются все CAN-пакеты, триггеры, ошибки, этапы прошивки

## 10. Дополнительные улучшения
- [x] Горячие клавиши (Ctrl+O, Ctrl+M, Ctrl+T, Esc)
