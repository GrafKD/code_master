# 📘 Руководство пользователя «Код Мастер»

Данное руководство поможет вам начать работу с приложением **Код Мастер** и использовать все его возможности.

---

## 📥 Установка

### Windows (портативная версия)

1. Перейдите на страницу [Releases](https://github.com/GrafKD/code_master/releases).
2. Скачайте последний релиз. Это может быть:
   - **Стабильный релиз** — `Release v1.x.x`.
   - **Тестовая сборка** — `Nightly build ...` (pre-release).
3. Распакуйте архив.
4. Запустите `CodeMaster.exe` двойным кликом.

> ⚠️ Приложение портативное. Оно не требует установки, прав администратора и не оставляет записей в реестре. Настройки сохраняются в файл `config.json` рядом с программой.

### Сборка из исходников

Если вы хотите запустить приложение из Python-кода:

```bash
git clone https://github.com/GrafKD/code_master.git
cd code_master
pip install -r requirements.txt
python main.py
```

Требования: Python 3.10+, PySide6, pyserial.

## 🔌 Первое подключение

После запуска приложения вы увидите главное окно. Для работы с STM32 или CAN-шиной нужно подключиться к COM-порту.

### Порядок действий

1. Нажмите кнопку **Настроить** (или `Ctrl+O` / `Ctrl+M` для быстрого открытия окна).
2. В диалоге выбора COM-порта:
   - Выберите порт из списка (например, `COM3` на Windows или `/dev/ttyUSB0` на Linux).
   - Укажите скорость (обычно `115200`).
   - При необходимости включите **Режим эмуляции** — приложение создаст виртуальный порт для тестирования.
   - Включите **Автопереподключение**, если хотите, чтобы приложение само восстанавливало связь при разрыве.
3. Нажмите **Подключить**.

Индикатор порта внизу окна станет зелёным, если подключение успешно.

## 🔥 Прошивка STM32

Приложение прошивает микроконтроллеры STM32 через встроенный UART bootloader.

### Пошаговая инструкция

1. Подключите STM32 к компьютеру через UART-USB адаптер и переведите его в режим bootloader (замкните BOOT0 на VCC, нажмите RESET).
2. В главном окне нажмите **Обновить** (или `Ctrl+O`).
3. Выберите файл прошивки `.bin` или `.hex`.
4. Нажмите **Открыть** — начнётся процесс прошивки.
5. Наблюдайте за прогресс-баром. После успешного завершения появится сообщение.
6. Переведите микроконтроллер в рабочий режим (отключите BOOT0 от VCC, нажмите RESET).

### Если прошивка не идёт

- Убедитесь, что порт открыт и устройство отвечает.
- Проверьте правильность скорости (обычно 115200).
- Попробуйте переподключить питание STM32 и повторить.
- Посмотрите логи — кнопка **📄 Логи** в главном окне.

## ⚡ CAN Тригеры

Тригеры позволяют автоматически отвечать на определённые CAN-кадры.

### Как настроить

1. Откройте окно настроек и перейдите на вкладку **CAN Тригер**.
2. В одном из блоков (их 10) заполните:
   - **ID** — CAN-идентификатор, на который реагирует триггер.
   - **D0–D7** — данные для фильтрации (можно оставить пустым, тогда сработает по любым данным).
   - **Активен** — включить триггер.
   - **ID отв.** — идентификатор ответного кадра.
   - **D0–D7 отв.** — данные ответного кадра.
   - **Задержка мс** — пауза перед отправкой ответа (0 — мгновенно).
3. Нажмите **Применить**.
4. Для проверки нажмите **Тест** — ответный кадр будет отправлен вручную.

> 💡 Каждый триггер имеет счётчик срабатываний. Он отображается рядом с чекбоксом «Активен».

## 📊 CAN Мониторинг

Мониторинг позволяет видеть весь CAN-трафик в реальном времени.

### Как запустить

1. Откройте вкладку **CAN Мониторинг** в окне настроек.
2. Для каждого канала (CAN1 / CAN2) нажмите **Старт**.
3. Входящие кадры будут появляться в таблице с отметкой времени, ID, данными и DLC.

### Полезные функции

- **Фильтр ID** — показывать только кадры с нужным идентификатором.
- **Приостановить** — временно остановить обновление таблицы.
- **Очистить** — очистить таблицу.
- **Отправить** — вручную отправить кадр с указанным ID и данными.
- **Циклически** — автоматически повторять отправку кадра с заданным интервалом.
- **Запись в CSV** — сохранить весь принятый трафик в CSV-файл.
- **Контекстное меню** — клик правой кнопкой по строке позволяет скопировать ID или данные, а также создать триггер на основе кадра.

## 🔄 CAN Шлюз

Шлюз позволяет перенаправлять кадры между каналами и заменять их содержимое.

### Настройка

1. Откройте вкладку **CAN Шлюз**.
2. В разделе **Игнорируемые ID** введите ID, которые не нужно пересылать.
3. В разделе **Правила замены** добавьте пары:
   - **Входной ID** — кадр, который пришёл.
   - **Выходной ID** — кадр, который уйдёт вместо него.
4. Нажмите **Применить**, затем **Старт**.

> ⚠️ Для работы шлюза оба канала должны быть активны.

## 🩺 Диагностика bootloader

Для быстрой проверки связи с STM32 bootloader:

1. Подключите устройство к COM-порту.
2. В главном окне нажмите **Диагностика**.
3. Приложение запросит версию bootloader и ID устройства.
4. Результат отобразится в окне сообщения.

Если устройство не отвечает, проверьте:
- правильность выбора порта;
- что устройство находится в режиме bootloader;
- наличие связи по UART.

## 🔧 Решение проблем

### Порт не виден в списке

- Убедитесь, что адаптер подключён к компьютеру.
- Проверьте драйверы USB-UART адаптера (CH340, CP2102, FTDI и т.д.).
- Перезапустите приложение.

### Устройство не отвечает при прошивке

- Проверьте, что STM32 находится в режиме bootloader (BOOT0 → VCC, RESET).
- Убедитесь, что скорость порта совпадает с настройками bootloader (обычно 115200).
- Проверьте линии RX/TX — они должны быть перекрещены (RX платы → TX адаптера, TX платы → RX адаптера).
- Проверьте общий GND.

### Ошибка синхронизации bootloader

- Убедитесь, что устройство действительно STM32 с поддержкой UART bootloader.
- Проверьте, что bootloader разблокирован (если включена защита от чтения, прошивка может не идти).
- Попробуйте снизать скорость до 9600 и повторить.

### Приложение не запускается

- Для Windows: убедитесь, что у вас Windows 10 или 11 (64-бит).
- Для разработчиков: проверьте, что установлены все зависимости из `requirements.txt`.
- При запуске из Python может потребоваться виртуальное окружение:

```bash
python -m venv .venv
source .venv/bin/activate  # Linux/macOS
# .venv\Scripts\activate   # Windows
pip install -r requirements.txt
python main.py
```

### CAN-кадры не приходят

- Убедитесь, что CAN-модуль подключён к выбранному COM-порту.
- Проверьте скорость CAN-моста и скорость COM-порта.
- Попробуйте запустить мониторинг на обоих каналах.
